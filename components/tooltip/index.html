Tooltip

Overview

A tooltip is a user-triggered floating text area that provides helpful or contextual information.

Sample component

Component status

Style

A tooltip can be placed on a light or dark background. It consists of text wrapped in a container that sometimes includes a subtle drop shadow. The container also includes an arrow that points to a Help or Information icon.

Light theme

Dark theme

Usage

Content

Content in a tooltip is limited to text only and should be brief and helpful to a user. If content needs to be more thorough or interactive elements need to be included, use a Popover or Modal instead.

Icon pairing

A tooltip should always be paired with a Help or Information icon, or be connected to a link or other component.

Orientations

Depending on the amount of content or available screen space, a tooltip can be used in different orientations. There is no default, but when choosing an orientation, be mindful of how a tooltip can potentially cover up important content or get cut off by the browser window.

Behavior

How a user interacts with a tooltip depends on what the screen size is.

  • On large screens, a tooltip disappears after a user moves their cursor off of the icon
  • On large screens, a tooltip disappears after the focus indicator is moved away from the icon
  • On small screens, a tooltip disappears after a user taps the icon again or taps outside of the tooltip

Responsive design

A tooltip can work on both large and small screens, depending on the amount of content or available screen space.

Breakpoints

A tooltip mostly remains the same on large and small screens.

Large screens

Small screens

Interaction states

The only interaction state is appearing next to an icon when focused, hovered, or tapped.

A tooltip can appear when hovered or tapped

Best practices

Accessibility

For better accessibility, do not use a light tooltip in a light environment because it will blend into the background too much.

Orientation

Content inside of a tooltip should not be cut off by the browser window, change the tooltip orientation if it does.

Pairing

Do not add a tooltip to things that do not require further explanation, like simple links and components.

Spacing

A tooltip uses PatternFly 4 spacers to define spacing values between elements. Every tooltip orientation contains the same amount of container padding and exterior spacing in between the arrow and icon.
